# DELTA — What's New vs. C.A.R.S.

## Overview

C.A.R.S. (Collaborative Affective Realtime Sim) is a browser‑based 2D molecular simulation where digital molecules drift, react, and bond via emotional chemistry. **BONDING** is its meatspace equivalent—but it's not a simulation; it's a facilitator.

---

## Key Differences

| Aspect              | C.A.R.S. (Digital)                              | BONDING (Meatspace)                           |
| ------------------- | ----------------------------------------------- | --------------------------------------------- |
| **Environment**     | Browser, 2D canvas                              | Real world, mobile app                        |
| **Interaction**     | Click / drag molecules                          | Physical presence, check‑ins, pings           |
| **Bonds**           | Simulated chemical bonds                        | Real human relationships                      |
| **Reactions**       | Visual animations                               | Real‑world tasks (helping, building, sharing) |
| **Zones**           | Abstract zones (Calm, Lab, Kitchen, Deep, Wild) | Physical places mapped to GPS                 |
| **Era progression** | Game time milestones                            | Cumulative real‑world impact                  |
| **Valence**         | Atom property                                   | Hidden reputation scalar (0.1–2.0)            |
| **Identity**        | Anonymous cookie                                | WebCrypto ephemeral keys                      |
| **Data**            | All on server                                   | Geohash‑only, witness‑signed proofs           |
| **Engagement**      | Simulation loop                                 | Human‑driven, off‑platform                    |

---

## What We Kept

- **The zone metaphor** (Calm, Lab, Kitchen, Deep, Wild).
- **Valence** as a measure of trust/chemistry.
- **Reaction** as the core event (bond formed, problem solved).
- **Era** as a sign of community maturity.

## What We Added

- **Physical location proof** (geohash + witness signatures).
- **Mutual opt‑in handshake** (Ping → Accept → Bond).
- **Reputation‑based matching** (complementarity, not just popularity).
- **Hyperlocal rollout** (launch in one neighborhood, not globally).

## What We Removed

- **Continuous GPS tracking** — only check‑in based.
- **Leaderboards and points** — no gamification of connection.
- **Variable‑ratio rewards** — fixed, predictable feedback.
